The Journey

After 30 years, I learned to code to see if I could build something better

After three decades working offshore in the North Sea, Middle East, Asia-Pacific, and West Africa, I kept seeing the same problems: brilliant engineers making critical decisions with incomplete data, multi-specialist conflicts causing costly delays, and vendors pitching PowerPoint solutions that never quite worked in the field.

So I taught myself to code and built the Brahan Engine - not as a theoretical exercise, but as an honest working prototype that demonstrates what AI-augmented governance could look like for offshore operations.

This is not a bootcamp portfolio. This is production infrastructure ($47/month, 99.97% uptime) that combines deep domain expertise with modern full-stack engineering.

What I Built

The Brahan Engine

An AI-augmented governance system for offshore operations that addresses the biggest cause of Non-Productive Time (NPT): multi-specialist conflict resolution under uncertainty.

Technical Stack:

  • React 18 frontend with modern hooks and Framer Motion
  • FastAPI backend on Google Cloud Run (serverless)
  • Firestore for real-time data, BigQuery for analytics
  • 30+ Playwright E2E tests for comprehensive coverage
  • Data Confidence Index (DCI) scoring engine
  • HSE safety veto controls and mission-critical logging

What makes it different:

  • Real production infrastructure, not a demo
  • Built by someone who's lived the problems for 30 years
  • Combines rare domain knowledge with modern engineering
  • Self-funded at $47/month (cost optimization mindset)

Why I Built This

Three decades offshore taught me that the hardest problems aren't technical - they're about making defensible decisions when you have incomplete information and multiple specialists disagreeing.

I built the Brahan Engine to demonstrate that AI can augment (not replace) human decision-making by:

  • Quantifying data confidence in real-time
  • Surfacing multi-specialist conflicts before they cause delays
  • Maintaining an immutable audit trail for forensic analysis
  • Enforcing safety vetoes while preserving operational momentum

This portfolio proves I can not only understand complex problems from 30 years of field experience, but also build production-quality software to solve them.
