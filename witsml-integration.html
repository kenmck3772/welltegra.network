WITSML/ETP Integration

Connect Well-Tegra with industry-standard data protocols for seamless well data exchange

WITSML v1.4.1.1 – v2.1 Real-Time ETP v1.2 Enterprise-Grade Security

Why WITSML/ETP Integration?

Industry Standard

WITSML and ETP are universally adopted protocols for well data exchange, supported by major operators, service companies, and software vendors worldwide.

Vendor Neutral

Avoid vendor lock-in with open, standardized data formats. Connect Well-Tegra to any WITSML-compliant system without proprietary adapters.

Real-Time Data

ETP enables streaming data from wellsite to office instantly. Monitor 20+ active rigs simultaneously with sub-second latency.

Data Quality

Built-in validation ensures consistent, high-quality data ingestion. Automatic schema validation, unit conversion, and quality scoring.

Supported Standards

WITSML

Versions: v1.4.1.1, v2.0, v2.1

Batch import/export of well data objects

  • ✓ Well: Header information, location, status
  • ✓ Wellbore: Trajectory, geometry, completion
  • ✓ Log: Time and depth-based logs
  • ✓ Trajectory: Survey data, inclination, azimuth
  • ✓ Risk: Operational risks, NPT tracking
  • ✓ Message: Daily reports, operational messages
  • ✓ + 8 more object types

ETP (Energistics Transfer Protocol)

Version: v1.2

Real-time WebSocket-based data streaming

  • ✓ Discovery: Browse available data objects
  • ✓ Store: Read/write data objects
  • ✓ Streaming: Real-time data subscription
  • ✓ Notification: Change notifications
  • ✓ Data Array: Efficient log/time-series transfer

Integration Capabilities

1. Data Ingestion

Real-Time Streaming (ETP)

  • • Subscribe to live drilling data feeds
  • • Automatic well selection and filtering
  • • Real-time data quality validation
  • • Configurable sampling rates (1-60s)
  • • Automatic reconnection on interruption

Batch Import (WITSML)

  • • Import historical well data
  • • Bulk import of multiple wells
  • • Incremental updates only
  • • Scheduled imports (hourly/daily/weekly)
  • • Detailed validation error reporting

2. Data Export

Push to WITSML Store

  • • Export analysis results to external stores
  • • Intervention plans as Risk objects
  • • Daily reports as Message objects
  • • Custom data mappings

ETP Publishing

  • • Publish Well-Tegra insights to subscribers
  • • Real-time analysis results streaming
  • • AI recommendation notifications
  • • Alert/alarm publishing

3. Bidirectional Sync

  • • Keep Well-Tegra in sync with corporate WITSML stores
  • • Automatic conflict resolution
  • • Change tracking and audit trails
  • • Rollback capabilities for data integrity

Quick Start API Example

Import from WITSML Store

POST /api/v1/integrations/witsml/import Authorization: Bearer YOUR_API_KEY Content-Type: application/json { "connection_id": "corp-witsml-prod", "wells": ["uid-1234", "uid-5678"], "objects": ["well", "wellbore", "log"], "validation": true }

ETP Real-Time Subscription

const etpClient = new ETPClient({ url: 'wss://etp.yourcompany.com/api/etp', applicationName: 'Well-Tegra', applicationVersion: '23.0.8' }); await etpClient.connect(); await etpClient.subscribeToChannels([ 'well/W666/log/GR', 'well/W666/log/ROP' ]);
